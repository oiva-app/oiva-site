This page will introduce the Oiva system and explain the high-level interaction between alerts, agents, telemetry, repositories, and Slack reports.

## Core Flow

An alert arrives through a webhook, Oiva launches an investigation, and the team receives live updates plus a final evidence-backed report.

## Primary Inputs

The first implementation integrates with observability data and Git repository context. Final copy will clarify which inputs are required and which are planned.

## Primary Outputs

Oiva's output is a structured incident report designed to be useful to humans, not a hidden internal agent trace.
